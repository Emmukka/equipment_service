# Техническое обслуживание и ремонт оборудования
Проект выполнен в соответствии с вариантом 5 — «Техническое обслуживание и ремонт оборудования».

Целью работы является разработка веб-ориентированной информационной системы для планирования и учета технического обслуживания и ремонтов производственного оборудования на предприятии.

Система предназначена для автоматизации процесса формирования планово-предупредительных ремонтов и представления результатов в виде календарного плана-графика.

# Описание предметной области
На предприятии используется разнотипное производственное оборудование, которое в процессе эксплуатации требует регулярного технического обслуживания и ремонтов. Работы выполняются в соответствии с нормативами завода-изготовителя.

В рамках данной предметной области выделяются следующие виды работ:
- Технический осмотр (ТО) — проводится регулярно с малым интервалом времени;
- Технический ремонт (ТР) — выполняется реже, чем ТО;
- Капитальный ремонт (КР) — проводится с наибольшим временным интервалом.

Периодичность работ является кратной друг другу. Например:
- ТО — 1 раз в месяц;
- ТР — 1 раз в 6 месяцев;
- КР — 1 раз в 3 года.
  
# Назначение и функции системы
Разрабатываемая информационная система предназначена для администратора предприятия и выполняет следующие функции:
- ведение реестра оборудования;
- хранение информации о типе оборудования;
- автоматическое формирование планов технического обслуживания и ремонтов;
- отображение календарного плана-графика работ;
- разграничение видов работ (ТО, ТР, КР);
- централизованный доступ через веб-интерфейс.

# Установка и запуск

1. Клонировать репозиторий:

git clone https://github.com/Emmukka/equipment_service.git
cd equipment_service

2. Создать виртуальное окружение и установить зависимости:

python -m venv venv
venv\Scripts\activate
pip install django==5.2.9 python-dateutil

3. Применить миграции:

python manage.py migrate

4. Создать суперпользователя (для доступа к admin):

python manage.py createsuperuser

5. Запустить сервер разработки:

python manage.py runserver

6. Перейти в браузере по адресу http://127.0.0.1:8000/ для доступа к Dashboard или http://127.0.0.1:8000/admin/ для администрирования.
